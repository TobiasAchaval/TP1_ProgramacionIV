API OpenWeatherMap

Demostración Práctica en Vivo

Solicitando permiso de ubicación para mostrar el clima...

Pronóstico para las próximas horas

Funcionalidad

Ofrece datos meteorológicos actuales, pronósticos por hora y día, humedad, viento, etc.

Autenticación

Requiere una API key gratuita que se obtiene al registrarse. Se envía como un parámetro en la URL, lo cual es un método simple.

Documentación

La documentación oficial es completa y se encuentra en su página web, con ejemplos claros para cada endpoint.

Formato de Datos

Los datos se devuelven en formato JSON, que es ligero y fácil de procesar con JavaScript.

Ejercicios Propuestos por Nivel de Complejidad

Objetivo: Crear una aplicación que muestre la temperatura y el estado del clima actual de una ciudad.
Habilidades a practicar: Realizar una solicitud HTTP GET con un parámetro de consulta, entender la estructura JSON de la respuesta y extraer datos específicos como la temperatura (`main.temp`) y la descripción del clima (`weather[0].description`).

Objetivo: Desarrollar una aplicación que muestre el pronóstico del clima para los próximos 5 días, incluyendo la temperatura máxima y mínima para cada día.
Habilidades a practicar: Manejar un conjunto de datos más grande, iterar sobre un array (`list`) en la respuesta JSON para extraer múltiples puntos de datos y agrupar la información para un resumen diario.

Objetivo: Construir una aplicación que envíe alertas personalizadas basadas en el clima (ej. "¡Llevar paraguas! Va a llover en 2 horas").
Habilidades a practicar: Combinar datos de pronóstico con lógica de programación para generar notificaciones.

API The Sports DB

Demostración Práctica en Vivo

Buscando el partido más importante de la historia...

Funcionalidad

Ofrece datos sobre equipos, jugadores, eventos, ligas y resultados de partidos.Permite buscar por deporte, liga o equipo.

Autenticación

Su versión gratuita no requiere clave para la mayoría de funciones, lo que la hace ideal para principiantes. La documentación menciona una clave gratuita ('123') que se puede usar en la URL.

Documentación

La documentación oficial está disponible online y detalla todos los endpoints disponibles para consultar la información deportiva.

Formato de Datos

Los datos se entregan en formato JSON, estándar para el intercambio de información en APIs modernas.

Ejercicios Propuestos

Objetivo del Proyecto: Construir una herramienta donde un usuario pueda buscar un equipo deportivo por nombre y ver detalles como su liga, estadio y logo.
Habilidades a Practicar: Manejar query parameters en la URL de la solicitud para la búsqueda, procesar el JSON y mostrar la información relevante del equipo.

Objetivo del Proyecto: Crear una página que muestre las estadísticas de un jugador específico de un equipo (ej. posición, fecha de nacimiento, etc.).
Habilidades a Practicar: Trabajar con endpoints dinámicos que requieren un ID de jugador, realizar múltiples llamadas a la API (primero para el equipo, luego para los jugadores de ese equipo) y filtrar datos dentro de la respuesta JSON.

Objetivo del Proyecto: Desarrollar un calendario interactivo que muestre los partidos de una liga con sus resultados o fechas futuras.
Habilidades a Practicar: Realizar múltiples solicitudes a la API, combinar la información de diferentes endpoints (equipos, resultados, eventos) y manejar la presentación de datos en un formato de calendario.

API The Movie Database (TMDb)

Demostración Práctica: Películas Populares del Momento

Cargando películas populares...

Funcionalidad

Es una base de datos masiva de películas y series. Permite buscar información detallada sobre títulos, actores, directores, géneros, calificaciones, tráileres y pósteres.

Autenticación

Requiere un "token de portador" (Bearer Token) que se obtiene al crear una cuenta. Este token se incluye en el encabezado de la solicitud HTTP, un método muy común en la industria.

Documentación

La documentación de la API de TMDb es muy completa y está disponible en su sitio web, detallando todos los endpoints y parámetros necesarios.

Formato de Datos

Todos los datos de la API se entregan en formato JSON, lo que facilita su integración con cualquier lenguaje de programación.

Ejercicios Propuestos

Objetivo del Proyecto: Desarrollar una aplicación que permita buscar películas por título y muestre el póster, el título y una sinopsis.
Habilidades a Practicar: Realizar la solicitud HTTP GET con un token de portador (Bearer Token), entender el manejo de imágenes a través de la API y mostrar la información básica de la película.

Objetivo del Proyecto: Construir un catálogo que muestre las películas más populares del momento, con detalles como el reparto, la calificación y el tráiler.
Habilidades a Practicar: Manejar peticiones paginadas para obtener más resultados, realizar una segunda solicitud a un endpoint diferente para obtener detalles específicos de una película y mejorar la presentación de datos complejos.

Objetivo del Proyecto: Construir una aplicación que permita al usuario explorar películas con filtros avanzados por género, año de estreno o calificación.
Habilidades a Practicar: Consumir la API con múltiples query parameters para filtrar los datos, construir la URL de forma dinámica y renderizar la interfaz de usuario con datos que cambian según los filtros.

Mejores Prácticas al Trabajar con APIs

Límites de Solicitudes

Las APIs gratuitas suelen tener límites de solicitudes por día. Es crucial leer la documentación para evitar superar estos límites y recibir un error 429 (Too Many Requests).

Manejo de Errores

El código debe estar preparado para fallos, como una clave de API incorrecta (error 401) o que el servidor de la API no funcione (errores 5xx), mostrando siempre un mensaje claro al usuario.

Seguridad de la API Key

La clave de la API nunca debe exponerse en el código del frontend en un proyecto de producción. La práctica correcta es almacenarla de forma segura en un servidor (backend).
